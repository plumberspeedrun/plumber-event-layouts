# Graphics for game
ゲーム映像と組み合わせて使用するgraphicsの共通仕様

## 画面構成
すべてのゲーム映像用graphicsで共通する画面構成部品は以下の通り。

- ロゴ画像
- 基本情報パネル (ゲームタイトル、リリース年、カテゴリー名、EST時間)
- 走者情報表示領域 (名前、各種SNSアカウントID)
- 解説情報表示領域(名前、各種SNSアカウントID)
- ゲーム映像表示領域
- 会場カメラ映像表示領域

### ロゴ
- Assetを参照する。

### 基本情報パネル
- 各種情報は`nodecg-speedcontrol`のReplicantを参照し、取得する。

### 走者情報表示領域
- 各種情報は`nodecg-speedcontrol`のReplicantを参照し、取得する。
- 指定秒数ごとに名前と各種SNSアカウントIDをフェードアニメーションで切り替える。
- 「名前」、「各種SNSアカウントID」の順番で切り替える。
(例: `name` -> `social.youtube`->`social.twitch`)
- 全プレイヤーの名前を同じタイミングで表示する。
- プレイヤーごとにSNSアカウントIDの種別数が異なる場合、最も種別数の多いプレイヤーの情報が１巡するタイミングで、全プレイヤーがループの最初に戻ること。（名前を表示する）

### 解説情報表示領域
走者名の代わりに解説者名を表示する。
その他は走者情報領域と同一の仕様。

### ゲーム映像表示領域
OBS Studio上でGraphics用ブラウザソースよりも下層に映像ソースを配置する。
仕組みは以下の通り。
- clip-pathで該当部分をくり抜き透過する。
- 透過部分に映像ソースを配置する。
